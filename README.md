        <h1 style="font-size: 28px; margin-bottom: 20px;">Low Cost ECG Device with Web-based Telemedicine</h1>
    </header>
    <div style="max-width: 800px; margin: 20px auto; padding: 20px; background-color: #fff; box-shadow: 0 0 10px rgba(0, 0, 0, 0.1); border-radius: 5px;">
        <p style="font-size: 16px; line-height: 1.6;">In today's healthcare landscape, accessibility is a critical concern. Our project addresses this challenge by introducing an affordable electrocardiogram (ECG) device paired with a web-based telemedicine platform. This innovation allows healthcare providers to remotely analyze ECG records of patients and provide diagnosis.</p>
        <h2 style="font-size: 20px; margin-bottom: 10px;">Key Components</h2>
        <ul style="margin-top: 10px; padding-left: 20px;">
            <li style="margin-bottom: 5px;">Hardware:
                <ul style="padding-left: 20px;">
                    <li>AD8232 sensor</li>
                    <li>ESP32S microcontroller</li>
                </ul>
            </li>
            <li style="margin-bottom: 5px;">Software:
                <ul style="padding-left: 20px;">
                    <li>Arduino framework for device programming</li>
                    <li>PHP for backend server management</li>
                    <li>Javascript for frontend browser interface</li>
                    <li>HTML5/CSS for web layout and styling</li>
                    <li>WS_FTP for secure file transfer</li>
                </ul>
            </li>
        </ul>
        <h2 style="font-size: 20px; margin-top: 20px; margin-bottom: 10px;">How It Works</h2>
        <div style="display: flex; align-items: center; margin-bottom: 20px;">
            <img src="https://github.com/animannaskar/Low-Cost-ECG-Device-with-Web-based-Telemedicine/assets/143376315/c3d4e467-4c84-472c-83da-f360f53c9ea5" alt="How It Works Image" style="max-width: 200px; margin-right: 20px;" align="right">
            <ol style="font-size: 16px; line-height: 1.6; flex-grow: 1;">
                <li>The device connects to the user's WiFi network.</li>
                <li>ECG signals are recorded by the device.</li>
                <li>The device uploads the ECG data to the server.</li>
                <li>The server processes the data and sends it to the client (healthcare provider).</li>
                <li>The ECG data points are plotted in the client's browser for analysis and diagnosis.</li>
            </ol>
        </div>
